# Retro: CI Failures — device-protocol/lib/messages_pb.js

## Timeline

| Run | Branch | Error | Root Cause |
|-----|--------|-------|------------|
| Pre-PR #45 | develop | SUCCESS (false) | No messages_pb check — shipped broken Linux bundles silently |
| PR #45 merge | develop | FATAL: messages_pb.js MISSING | Fail-fast check exposed existing bug |
| CI fix #1 | release/1.2.6 | `protoc: not found` | CI runner has no protoc installed |
| CI fix #2 | release/1.2.6 | `protoc-gen-js: not found` | apt protoc doesn't include JS plugin (deprecated since protobuf v21) |

## Root Cause

`modules/device-protocol` has `lib/` in `.gitignore`. The compiled protobuf
files (`messages_pb.js` etc) are build artifacts generated by `protoc` with
the `protoc-gen-js` plugin. This plugin was deprecated and removed from the
protobuf distribution in v21 (2022). The Ubuntu runner's protobuf-compiler
package ships v21+ which doesn't include it.

## Why Old CI "Worked"

Old `collect-externals.ts` had no verification. It copied whatever was in the
`device-protocol` directory — including an empty `lib/` with just a `.keep`
file. The Linux build completed and the AppImage was published, but it would
crash on launch with `Cannot find module 'device-protocol/lib/messages_pb'`.
Nobody caught this because the Linux AppImage was never tested with a real
KeepKey device.

## Why Building From Source Doesn't Work

1. `protoc-gen-js` is deprecated — not in modern protobuf packages
2. `build:postprocess` uses `sed -i ''` (macOS BSD sed) — fails on Linux/Windows
3. Even if protoc worked, the build chain is fragile and macOS-specific

## Fix Options

### Option A: Copy lib/ from hdwallet's resolved dependency (RECOMMENDED)

After `yarn install` in hdwallet, `modules/hdwallet/node_modules/@keepkey/device-protocol/lib/`
contains the pre-built files. Copy them into the submodule's `lib/` directory.

Pros: No protoc needed, uses files that hdwallet already validated
Cons: Depends on hdwallet build running first (it already does)

### Option B: Check in lib/ to the device-protocol repo

Remove `lib` from `.gitignore`, commit the built files.

Pros: Simplest, always available on checkout
Cons: Requires push to device-protocol repo, generated files in git

### Option C: Publish device-protocol to npm with lib/ included

Use the npm-published version instead of `file:` link.

Pros: Standard npm resolution, lib/ included in published package
Cons: Requires npm publish workflow, version coordination

## Recommendation

Option A for immediate fix. It's the same approach the Windows agent used
in ALPHA-6 to get the build working. The build order is already correct
(hdwallet builds before vault), so the files are guaranteed to exist.
